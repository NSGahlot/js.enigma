title: DOM API
---

1. **Tag Name Selector**:
   - Selects elements by their tag name (e.g., "div", "p", "h1")
   - Example: `document.getElementsByTagName('div')`

2. **ID Selector**:
   - Selects an element with a specific ID
   - Example: `document.getElementById('my-element')`

3. **Class Selector**:
   - Selects elements with a specific class name
   - Example: `document.getElementsByClassName('my-class')`

4. **Universal Selector**:
   - Selects all elements on the page
   - Example: `document.querySelectorAll('*')`

5. **Descendant Selector**:
   - Selects elements that are descendants of a specified parent element
   - Example: `document.querySelectorAll('div p')`

6. **Child Selector**:
   - Selects elements that are direct children of a specified parent element
   - Example: `document.querySelectorAll('ul > li')`

7. **Adjacent Sibling Selector**:
   - Selects elements that are adjacent siblings of a specified element
   - Example: `document.querySelectorAll('h1 + p')`

8. **Attribute Selector**:
   - Selects elements based on their attribute values
   - Examples:
     - `document.querySelectorAll('a[href]')`
     - `document.querySelectorAll('input[type="text"]')`

9. **Pseudo-class Selector**:
   - Selects elements based on their state or position
   - Examples:
     - `document.querySelectorAll('a:hover')`
     - `document.querySelectorAll('li:nth-child(even)')`

These selectors can be combined and nested to create more complex selection patterns. The `querySelector()` and `querySelectorAll()` methods are the most flexible and powerful ways to select DOM elements.

**The different DOM selectors return various data types, and in some cases, you may need to convert the result to an array for easier manipulation. Here's a breakdown:**

1. `document.getElementsByTagName()`, `document.getElementsByClassName()`:
   - Returns an HTMLCollection, which is a live, dynamic collection of elements.
   - To convert to an array, you can use `Array.from()` or the spread operator: 
   `const myElements = Array.from(document.getElementsByTagName('div'))`.

2. `document.getElementById()`:
   - Returns a single Element object, not a collection.

3. `document.querySelectorAll()`:
   - Returns a NodeList, which is a static, non-live collection of nodes.
   - NodeLists can be converted to an array using `Array.from()` or the spread operator: `const myElements = [...document.querySelectorAll('div')]`.

4. `document.querySelector()`:
   - Returns a single Element object, not a collection.

The key differences are:

- HTMLCollections are live, meaning they update automatically when the DOM changes. NodeLists are static.
- HTMLCollections cannot be directly converted to an array, but NodeLists can.

Typically, you'll want to convert the result to an array if you need to perform array methods like `map()`, `filter()`, or `forEach()` on the elements. This is especially important when working with HTMLCollections, as they can change unexpectedly during iteration.
